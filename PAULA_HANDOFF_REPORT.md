## What I Built

### 1. Output Generator — `backend/output_generator.py`

**Node:** N11  
**Step:** 7 in build plan  

One Anthropic API call that receives `ScoredOutput`, loads the correct communication framing for the patient's identity, calls the model, validates the response, and returns a `ChecklistOutput`.

**Key decisions:**
- Model: `claude-sonnet-4-20250514`, `max_tokens=2000`, no streaming
- Framing is loaded from `framing_library.json` using `{race_ethnicity}|{payer}` as the key — falls back to `"default"` if no exact match (covers "Other" payer and unsupported race values)
- `FramingBlock`, `conflict_flags`, and `clinical_disclaimer` are assembled in Python — not generated by the model. Only `items`, `hospital_status`, and `confidence_summary` come from the model
- AWHONN licensing rule enforced: validation rejects any `ChecklistItem` where `source` contains "awhonn" (case-insensitive)
- All `action` fields validated to begin with `"Consider screening for..."` before the `ChecklistOutput` is constructed
- Model response is stripped of markdown fences if the model wraps its output
- `AsyncAnthropic` client — works with the async orchestrator pipeline

**Validation rules enforced in code:**
- Every `ChecklistItem` must have a non-empty `source`
- No AWHONN in any source field
- All actions must begin with `"Consider screening for..."`
- `items` list cannot be empty
- `clinical_disclaimer` is hardcoded — never modified

**Environment variable required:**
```
ANTHROPIC_API_KEY=your_key_here
```

### 2. Input Form — `frontend/index.html`

**Node:** N1  
**Step:** 10 in build plan  

8-field CNM input form. Tailwind via CDN, no build step.

**Fields:**
| Field | Type | Notes |
|---|---|---|
| age | number input | required |
| race_ethnicity | dropdown | exact values per handoff spec |
| payer | dropdown | Medicaid / Private / Other |
| state | dropdown | NY and TX first (full output); all 50 states listed (partial output for others) |
| hospital_name | text input | required |
| weeks_postpartum | number input | required |
| complications_flagged | checkboxes (multi-select) | exact string values per handoff spec |
| primary_language | text input | required |

**Exact dropdown values used (as specified in handoff):**

Race / ethnicity:
- `"Black or African American"`
- `"White"`
- `"Hispanic or Latino"`
- `"Asian"`

Complications:
- `"hypertension"`, `"preeclampsia"`, `"hypertensive disorder"`, `"mental health"`, `"depression"`, `"anxiety"`, `"substance use"`, `"OUD"`

**Client-side validation:** checks all required fields before sending the request. Shows a field-level error message if anything is missing.

**Loading state:** submit button disables and changes text to "Generating checklist…" while waiting for the API response.

**Error display:** API errors, validation errors, and timeout messages render in a red banner above the form.

### 3. Checklist Display — `frontend/checklist.html` + `frontend/app.js`

**Node:** N13  
**Step:** 10 in build plan  

Reads `ChecklistOutput` from `sessionStorage` (set by `app.js` after a successful API response) and renders the full checklist.

**Display order (matches build plan spec exactly):**
1. Patient context header — state, race/ethnicity, payer, weeks postpartum, language, complications, urgency tier badge
2. Clinical context / framing block — communication framing copy with source links and AWHONN "see also" reference
3. Checklist items — all items ordered by `priority_rank` (1 = highest urgency), each showing label, detail, action, source, and confidence badge
4. Hospital commitment status — CMS Birthing-Friendly designation, HCAHPS discharge score, state PPC-AD rate
5. Conflict flags — shown only if present; purple styling, "CNM review required" label
6. Data confidence summary
7. Clinical disclaimer — hardcoded text from `ChecklistOutput.clinical_disclaimer`

**Confidence badge colors:**
- `H` — red
- `M` — yellow/amber
- `L` — green
- `FLAGGED` — purple

**Session handoff:** `app.js` stores both the `ChecklistOutput` JSON and the original patient payload in `sessionStorage` and redirects to `checklist.html`. If `sessionStorage` is empty when `checklist.html` loads, it redirects back to `index.html`.
